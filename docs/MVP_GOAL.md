## 1. MVP Objective

The first version of **X!Y — The Explorer Factory** is designed to validate the core marketplace model by enabling visionaries and manufacturers to discover each other, communicate, negotiate, and complete manufacturing requests end to end.

The MVP focuses exclusively on the critical path between demand (Visionaries / Ideators / Startups) and supply (Manufacturers / Factory Owners). Secondary ecosystem roles—vendors, labor suppliers, logistics providers, legal professionals, investors, and distribution partners—are intentionally deferred so the team can prove marketplace viability with minimal development complexity and clear learning outcomes.

Success for this release means confirming that users will engage with a structured digital workflow for manufacturing requests, and that completed engagements generate enough trust and value to support future expansion into a full multi-sided manufacturing ecosystem.

---

## 2. MVP Success Criteria

The MVP will be considered successful when users can complete the following core marketplace actions without relying on offline workarounds:

| Capability | Success Condition |
|------------|-------------------|
| **Register and authenticate** | Users can create accounts and sign in securely |
| **Create profiles** | Visionaries and manufacturers can establish complete profiles |
| **Discover manufacturers** | Visionaries can find relevant manufacturers via search and directory |
| **Search machinery** | Users can locate machinery by attributes and filters |
| **Submit manufacturing requests** | Visionaries can create and submit structured project requests |
| **Communicate within the platform** | Parties can message in the context of a specific request |
| **Negotiate offers** | Manufacturers can accept, decline, or counter; visionaries can respond |
| **Confirm bookings** | Both parties can finalize capacity bookings |
| **Track request progress** | Users can view status updates across the request lifecycle |
| **Complete manufacturing engagements** | Requests can move from submission to completion |
| **Leave reviews and ratings** | Users can submit feedback after engagement completion |

If these criteria are met at meaningful volume, the platform has demonstrated product–market fit for its core two-sided marketplace loop.

## 3. Visionary User Journey

The Visionary journey defines the demand-side MVP workflow—from account creation through feedback after a completed manufacturing engagement.

### Onboarding

- Sign up and log in securely
- Complete a personal / startup profile

### Discovery

- Search manufacturers using filters such as:
  - Location
  - Industry
  - Machinery type
  - Availability
  - Labor support
  - Logistics support
  - Pricing
  - Keywords
- View manufacturer profiles
- View machinery details, including:
  - Images
  - Specifications
  - Pricing
  - Availability
  - Services
  - Certifications (when available)

### Request & Collaboration

- Submit a manufacturing request
- Attach project information and relevant documentation
- Send messages within the request thread
- Receive manufacturer responses:
  - Accepted requests
  - Declined requests
  - Counter-offers
- Review counter-offers
- Accept manufacturer offers

### Booking, Tracking & Feedback

- Confirm bookings
- Track request status throughout the process
- Leave ratings and feedback after project completion

This journey validates that ideators can move from idea to booked manufacturing capacity entirely within X!Y.

---

## 4. Manufacturer User Journey

The Manufacturer journey defines the supply-side MVP workflow—from company onboarding through listing management, request handling, and post-engagement feedback.

### Onboarding & Listing Setup

- Register and log in
- Create a company profile
- Create a factory profile
- Add machinery listings with:
  - Specifications
  - Pricing
  - Service charges
  - Availability configuration
  - Labor information
  - Logistics services
  - Raw material support details
  - Machinery images
  - Insurance information
  - Keywords
- Submit listings for approval (if applicable)

### Request Handling

- Receive manufacturing requests
- View request details
- Chat with visionaries
- Accept requests
- Decline requests
- Send counter-offers

### Operations & Reputation

- Manage bookings
- Update availability
- Modify pricing
- Receive ratings and reviews

This journey validates that manufacturers can digitize spare capacity, attract demand, and close manufacturing engagements through the platform.

## 6. Out of Scope for MVP

The following capabilities are explicitly excluded from the MVP and planned for future releases. Deferring them protects delivery focus and avoids expanding scope beyond marketplace validation.

| Category | Deferred Features |
|----------|-------------------|
| **Ecosystem marketplaces** | Investor Marketplace, Vendor Marketplace, Labor Marketplace, Logistics Marketplace, Legal Services Marketplace |
| **Intelligence & automation** | AI-based recommendations, Smart Matching Engine, Predictive Analytics |
| **Platform expansion** | Mobile applications, International expansion, Multi-language support |
| **Monetization & growth** | Subscription billing, Advertisements, Advanced reporting |
| **Payments** | Payment gateway integration *(deferred if not required for initial booking confirmation)* |

Out-of-scope items may be reconsidered only if they become blockers to validating the core visionary–manufacturer workflow.

## 7. MVP Validation Metrics

The following metrics will measure whether the MVP is achieving its validation goals. Targets should be defined during launch planning and reviewed on a fixed cadence (e.g., weekly during soft launch, monthly thereafter).

| Metric | Purpose |
|--------|---------|
| **Number of registered users** | Overall platform adoption |
| **Number of manufacturers onboarded** | Supply-side growth and liquidity |
| **Number of machinery listings** | Depth of discoverable capacity |
| **Number of manufacturing requests created** | Demand-side engagement |
| **Request acceptance rate** | Match quality and manufacturer responsiveness |
| **Booking completion rate** | Conversion from request to confirmed engagement |
| **Average response time** | Speed of manufacturer–visionary collaboration |
| **User retention** | Sustained engagement beyond first use |
| **Ratings and reviews submitted** | Trust loop completion and feedback culture |
| **User satisfaction** | Qualitative validation of platform value |

Primary validation hinges on **request volume**, **acceptance rate**, and **booking completion rate**—indicators that both sides of the marketplace are successfully interacting through the product.

## 8. Expected MVP Outcome

Upon successful delivery and adoption of the MVP, X!Y will have demonstrated that:

- Visionaries can discover manufacturers.
- Manufacturers can showcase available capacity.
- Both parties can communicate and negotiate within the platform.
- Manufacturing requests can progress through a complete lifecycle.
- Bookings can be confirmed against available capacity.
- Feedback can be collected to build marketplace trust.
- The core marketplace business model is validated before expanding into additional ecosystem services.

The MVP is not intended to deliver the full X!Y vision of an end-to-end manufacturing ecosystem. It is intended to prove the foundational two-sided interaction—idea to capacity to completed engagement—that all future stakeholder marketplaces and intelligent features will build upon.
